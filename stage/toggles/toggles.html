Selection Controls

Description

Selection controls describes different methods for making selections in the LEAP products, including checkboxes, radio buttons, and switches.

Usage

Selection controls are used whenever a user needs to interact with a boolean state or a multi-select state without providing text input.

When to Use

  • This pattern should be used for selecting multiple options in a set (checkbox)
  • This pattern should be used for selecting one option out of a set (radio button and dropdown)
  • This pattern should be used for toggling an option on and off (switch)

When Not to Use

  • This pattern does not cover user text input

Detailed Interaction Description

Checkbox

Checkboxes allow a user to make multiple selections. They are also used as a boolean selector when a user needs to opt into a selection, such as “remember password”, “agree to terms and conditions”, etc. This is in contrast to switches where the default boolean state is “off”.

Radio Button

Radio buttons are used for making a single selection out of a set of options. It should be used when you need to display all the options side by side, otherwise a dropdown accomplishes a similar goal.

Switch

Switches toggle a state of a single setting, turning it on or off.

Visual Design Details

Design Considerations

The selection controls are used in a variety of ways and are not necessarily constrained to the style outlined in the Visual Design Details section.
