User Management

Description

User management covers a user's ability to add and remove participants from a project as well as modify their permission level within that project. This pattern refers to user management at the application level.

Usage

User management is used to invite new users as well as add and remove existing users to a project. It is also used to change permission levels of participants in a project. It can be used to view which users are currently participating on a given project.

When to Use

  • This pattern should be used when you need to add or remove users
  • This pattern should be used in applications that have different permission levels for users.
  • This pattern should be used in applications that have a subset of users participating in a subset of projects within the application.

When Not to Use

  • This pattern does not cover managing users at a subscription level.
  • This pattern does not cover managing users across applications.

Detailed Interaction Description

Selecting Roles

Roles are selectable with a dropdown. When the list of roles is short, no background color/border is used in the dropdown.

Search

The search field allows you to not only search for existing users in the system, but also invite new users via email to the application. When you enter an email that isn't recognized, you have the option of sending them an invite via email.

Invited users appear at the top of the list, making it easy to un-invite them or resend their invitation.

The information for a newly added user includes:

  • Email Address
  • Invitation status (pending)
  • Invite link for resending email invitation
  • Role selector

Removing Users

The system will surface an error if the last owner is being removed or if the owner has sections that belong to them. There must always be at least one owner in the document at all times and the owner must re-assign his sections before he can be removed.

Non Admin View

The non-admin view omits the role dropdown select as well as the ability to add/remove users. They have the ability to search through users in the application, but not all users on a given LEAP subscription.

Design Considerations

While user management takes the form of a modal in Concert, the pattern isn't limited to a modal. If needed, it can be expanded to a larger layout. The list of users is displayed alphabetically, with invited users shown at the top of the list.
