User Profile

Description

The user settings menu contains high-level information and options related to the current user. The menu is displayed when the user selects the option in the global navigation.

Usage

The User Settings Menu provides access to information and options that are global in nature and relate directly to the current user. The menu should be made accessible from a single location in the global navigation. The following basic information should always appear in the User Settings Menu:

  • Avatar
  • First and last name
  • Email address (sign-in)
  • My Apps link
  • Sign Out option (this option should always appear last)

Additional items may be included, depending on the need of the app. Like the base items, app-specific information and options should reflect the global context of the app.

When to Use

  • This pattern should be used in all LEAP apps to provide access to global user options and profile information

When Not to Use

  • This pattern should be used only in the global context. This User Settings Menu should not be displayed as a result of clicking a user avatar in a data grid, or user list.

Detailed Interaction Description

The user profile menu drops down from the user’s name in the global header to reveal application level settings.

Visual Design Details

Design Considerations

The options “My Apps” and “Sign Out” are default options in the user profile menu, but other items can be added as needed by an application.
