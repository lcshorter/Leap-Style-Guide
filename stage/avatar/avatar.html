User Avatars

Description

User avatars are the visual representation of users within an application. This pattern details the rules that govern user avatars as well as the content that appears within them.

Usage

User avatars are used to help people identify other users within the application. They may take the form of an uploaded portrait of the user or a colored fill with the user’s initials.

When to Use

  • This pattern is used anytime you need a visual representation of a user.
  • This pattern is used in the global header as well as in all user management within a project.

When Not to Use

  • This pattern does not cover managing users at a subscription level.

Detailed Interaction Description

Default Avatar

When a user is first invited to a project and has yet to accept their invitation to join the project, their avatar is a generic profile icon.

Assigned Color

When a user is in the system, but has yet to upload a profile picture for themselves, they are assigned a color from the system and their first and last initials are displayed within.

Visual Design Details

The avatar itself comes in two sizes. Large: 40x40

Small: 24x24

Design Considerations

The colors have been chosen with consideration to LEAP’s existing color palette along with it’s ability to be displayed on both light and dark backgrounds. If there are legibility concerns or a need for more colors contact the LEAP design team to inquire further about updating the colors.
