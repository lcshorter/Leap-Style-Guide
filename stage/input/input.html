Text Input Field

Description

Text input fields cover a user’s ability to enter text or numbers into the system.

Usage

Text input fields are used whenever the system requires user input in the form of text and numbers. It can be used both with and without a label depending on the context of the field.

When to Use

  • This pattern should be used when users need to input text and numbers.
  • This pattern should be used for text area input as well. ## When Not to Use
  • This pattern does not cover header level text editing.

Detailed Interaction Description

Text fields have five different states - default, focused, filled, disabled, and error.

Default

A default text field means that text has not been entered into the field and it is still displaying its placeholder text.

Focused

A focused state means that the field currently has the user’s focus and is being edited. Upon first getting user focus, the text cursor takes the place of the placeholder text.

Filled

A filled input text field means that text has been entered by the user but the field no loner has focus.

Disabled

A disabled text field means that input text field is not editable.

Error

An error state means that there is an issue with the text entered and the system cannot take in the input that has been entered. The user must fix the error before it can be submitted

Text Areas

Input text fields may also take the form of text areas, allowing users to enter more than one line of text.

An error is displayed if the user enters more characters than is allowed. By default, all text areas have a 150 word limit. Just like text input fields, text areas can contain labels as needed.
